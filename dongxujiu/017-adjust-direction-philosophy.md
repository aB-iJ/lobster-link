## 2. 纤维丛的具体定义（东旭的原话）

- **纤维**："相近义的量化指标维度距离与关系"
  - 我的理解：每个词的纤维是一个局部空间，包含它的近义词（同义词、变体）以及它们之间的语义距离
  - 比如"猫"的纤维里有 {猫咪, 小猫, 喵星人, ...} 以及它们两两之间的距离
- **联络**："纤维的分散和集群"
  - 我的理解：描述纤维如何分散成变体，又如何聚类成更抽象的语义群
  - 比如"猫"→"宠物"→"动物"的层级联络，或者"猫"→"狗"的类比联络

## 3. 成功标准降低到 10%

东旭的原话："创新和一般进步的区别在于创新是地基向上搭建、空中楼阁同时向下联通的过程，所以不合理性不应该被直接否定，而是合理化。"

**哲学**：创新往往是先有大胆框架（空中楼阁），再往下补地基（合理性）。所以：
- 10% 提升就算 positive signal，证明方向有潜力
- 不合理的部分不要直接否定，而是想办法合理化（比如 KLC 的 R3 规则）
- 这是**探索性研究**的心态，不是工程优化

## 4. 优先级：继续深挖几何结构

即使验证结果只有 10% 提升，也继续探索其他几何结构（乘积流形、动态曲率、李群等）。

## 5. 具体调整方案

### 工程上：
- **前端**：马上开始 Three.js + React 框架搭建，用 mock 数据
- **后端验证**：同时跑 geoopt 实验，出结果就接入
- **数据管道**：预训练嵌入 → 几何变换 → 纤维丛映射

### 检查点（Milestone）：
- **M1**（一周后）：前端能展示一个词的纤维（3D 点云 + 距离）
- **M2**（两周后）：后端出欧氏 vs 双曲 vs 球面的量化对比
- **M3**（三周后）：前后端打通，能实时切换几何空间看同一组词

### 风险管理：
如果 M2 结果完全不理想（0% 提升或负提升），我们暂停实验部分，但平台框架继续——可以作为通用几何可视化工具复用。

## 7. 建议的纤维丛形式化草案

我初步想的数学框架：

1. **总空间 E** = 底空间 B × 纤维 F
2. **底空间 B**：词的嵌入（欧氏/双曲/球面）
3. **纤维 F_x**：词 x 的近义词空间（局部几何）
4. **联络 Γ**：从一个纤维到另一个纤维的"平行移动"，编码语义变换
