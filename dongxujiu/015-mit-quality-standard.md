东旭补了一个要求：**参考 MIT 的交互式可视化网站，只能更好。**

也就是说这不是一个内部实验工具，而是一个**展示级别的交互式 Web 平台**。

## 参考标准

MIT 系的经典交互式可视化：

- **Immersion (MIT Media Lab)**：邮件社交网络的动态可视化
- **Tensorflow Playground**：神经网络的实时交互式训练可视化
- **Distill.pub 的文章**：可交互的数学/ML 概念解释
- **3Blue1Brown 风格**：优雅的数学动画 + 直觉构建

我们要做的应该至少达到这个水准：

### 视觉
- 干净、现代、学术感但不枯燥
- 3D 场景丝滑流畅（60fps）
- 配色和排版有设计感，不是 matplotlib 截图

### 交互
- 鼠标悬停看信息、点击选词、拖拽旋转空间
- 实时切换几何空间（欧氏 ↔ 双曲 ↔ 球面），看同一组词的嵌入如何变化
- 拉一条"语义路径"（比如从 cat 到 dog），看沿路的曲率变化
- 纤维丛的可视化：点一个词，展开它的纤维（多义空间）

### 叙事
- 不只是工具，有引导式的"故事线"
- 从"为什么欧氏空间不够"开始，逐步引入双曲、球面、纤维丛
- 每一步都有可交互的 demo

## 技术栈更新

考虑到展示品质要求：

- **3D 渲染**：Three.js + React Three Fiber（生态最好，效果最强）
- **UI 框架**：Next.js 或 Astro（SSG，快）
- **动画**：Framer Motion + GSAP
- **数学渲染**：KaTeX（实时公式）
- **数据计算**：后端 Python API（geoopt），前端调用
- **部署**：Vercel 或 GitHub Pages

PCL 就不用了——Web 端的 Three.js 在展示层面完全够用，而且任何人打开浏览器就能体验。
