## 已经有人在做的

### 1. 光学矩阵乘法（最成熟）

矩阵乘法是 LLM 训练/推理的核心瓶颈。光学方案：

- **Mach-Zehnder 干涉仪阵列**：光通过一系列分束器和移相器，干涉过程天然完成矩阵-向量乘法
- **代表公司**：Lightmatter（MIT 出来的）、Lightelligence
- 优势：**光速完成、几乎零能耗**（光子不产热）、天然并行
- MIT 2017 年的论文就演示了用光学芯片跑神经网络前向传播

### 2. 衍射光学神经网络（D²NN）

UCLA 的 Ozcan 团队（2018）：

- 用 3D 打印的**衍射层**堆叠，每层是一个相位掩模
- 光打进去，经过多层衍射，出来的光场分布就是分类结果
- **字面意义上的"光跑一遍就出结果"**——推理延迟是光传播时间（纳秒级）
- 能耗几乎为零（不需要电，只需要光源和探测器）

### 3. 光谱编码计算

这个和你说的**用光谱数据转译成结果**最接近：

- 不同波长编码不同信息通道
- 用波分复用（WDM）在单根光纤里同时跑成百上千个通道
- 检测端用光谱仪读出各波长强度 → 解码为计算结果

### 4. 你的"信息密度→几何体积"直觉在光学里的对应

| 你的想法 | 光学对应 |
|---|---|
| 不同点赋不同面积/体积 | 不同空间位置/波长的光强不同 |

| 信息密度编码 | 光场的振幅+相位天然是复数，比数字标量携带更多信息 |

## 现实瓶颈

1. **训练难**：光学系统擅长前向传播（推理），但反向传播（训练）很难在光域里做，目前还是要回到数字域训练，光域只做推理
2. **精度**：光学器件的制造误差 → 目前精度大约等效 4-8 bit，远低于 FP16/BF16
3. **非线性**：神经网络需要非线性激活函数，光学天然是线性的，加非线性需要额外器件（光电转换或饱和吸收体），会引入损耗和延迟
4. **规模**：目前演示的最大规模大概几千个参数，离 LLM 的千亿参数差好几个数量级

## 前沿动态

- **Lightmatter** 2024 年推出了 Passage 互连平台，用光学做芯片间通信，已经在和 GPU 厂商合作
- **NVIDIA 也在投光互连**，不是用光做计算，而是用光解决 GPU 之间的带宽瓶颈
- 学术界在探索**全光学训练**（in-situ training），但还很早期

## 总结

你的直觉方向是对的——**让物理过程本身当计算引擎，测量结果即输出**。这不是科幻，是正在发生的事。但目前光计算更适合做**推理加速**（特别是矩阵乘法密集的 Transformer 推理），离取代 GPU 做完整的 LLM 训练还有距离。
